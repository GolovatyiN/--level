## Старт

### Регистрация и вход
- Открой страницу `/auth`
- Кнопка «Уже есть аккаунт? Войти» — вход по email и паролю
- Кнопка «Нет аккаунта? Зарегистрироваться» — регистрация
- Минимальная длина пароля — **6 символов**
- Email-подтверждение **отключено** для удобства тестирования (можно включить в Supabase Dashboard)

### Первый вход
- При регистрации автоматически создаётся профиль и роль «Пользователь»
- Чтобы получить доступ к разделу «Управление» — попроси супер-админа поднять тебе роль до **Админа** или выше
- Свой профиль (имя) можно изменить в **Управление → Пользователи**

## Роли и права доступа

В системе пять ролей. Чем выше роль — тем больше прав.

| Роль | Что может |
|---|---|
| **Супер-админ** 👑 | Всё. Управляет ролями (включая других супер-админов), кварталами, отделами, любыми планами и задачами |
| **Админ** 🛡️ | Управляет пользователями, отделами, кварталами; утверждает квартальные планы; назначает любые роли кроме супер-админа |
| **Руководитель отдела** 🛡️ | Видит свои отделы. Создаёт и редактирует планы своих отделов, отправляет на согласование, отвечает на комментарии |
| **Менеджер** 👥 | Видит назначенные ему отделы. Может создавать и редактировать задачи в рамках доступа |
| **Наблюдатель** 👁️ | Только чтение |

### Полная таблица возможностей

| Возможность | Супер-админ | Админ | Рук. отдела | Менеджер | Наблюдатель |
|---|---|---|---|---|---|
| Просмотр данных | ✓ | ✓ | ✓ | ✓ | ✓ |
| Создание задач/целей | ✓ | ✓ | ✓ | ✓ | — |
| Редактирование | ✓ | ✓ | ✓ | ✓ | — |
| Удаление | ✓ | ✓ | — | — | — |
| Управление пользователями | ✓ | ✓ | — | — | — |
| Управление отделами | ✓ | ✓ | — | — | — |
| Назначение ролей | ✓ | — | — | — | — |
| Доступ ограничен отделами | — | — | ✓ | ✓ | ✓ |
| Журнал действий | ✓ | ✓ | — | — | — |

> Подробнее о ролях смотри в **Управление → Роли и права доступа**.

## Панель управления

Главный аналитический экран. **Не таск-менеджер** — здесь общая картина, детали раскрываются по клику.

В шапке:
- **Селектор квартала** — по умолчанию текущий квартал. «Все кварталы» показывает срез по всем периодам
- **Селектор отдела** — фильтрует все блоки на странице по отделу
- **Кнопка «Сброс»** — возвращает в дефолтный scope

### 1. Шесть карточек статусов задач (сверху)
- **Всего**, **В работе**, **Под риском**, **Блокеры**, **Завершено**, **Просрочено**
- Считаются с учётом квартала и отдела из шапки
- **Клик по карточке** → переход в «Планы и задачи» с пред-применённым фильтром

### 2. По приоритетам
Четыре карточки: **Критический**, **Высокий**, **Средний**, **Низкий**
- Каждая карточка показывает количество **активных** задач этого приоритета
- **Дельта за неделю** — `+N за неделю` (нагрузка растёт), `-N за неделю` (разгрузка), `без изменений`
- Тоггл «Показывать завершённые» — добавляет в подсчёт завершённые задачи
- **Клик** → `/tasks?priority=...` с применённым фильтром

### 3. Квартальные планы
Шесть карточек по статусам (Бэклог, В процессе, На согласовании, Нужны правки, Готов, Завершён) + средний прогресс по компании. Снизу — два списка:
- **Без плана на текущий квартал** — отделы, у которых ещё нет строки плана
- **Ожидают согласования** — клик ведёт прямо в план

### 4. По отделам
Карточки отделов с прогресс-баром, количеством задач, чипами «Просрочено», 🔥 (критические), «Высокие», «Средние».
- **Клик по карточке отдела** → открывает модалку с задачами этого отдела
- В модалке: статистика, фильтры (Все / Критические / Высокие / Средние / Просроченные / В работе / Завершённые / по направлениям), поиск, компактный список задач

### 5. Аналитика — четыре графика
- **Задачи по статусам** (donut)
- **Задачи по приоритетам** (donut)
- **Прогресс по отделам** (горизонтальный список с барами; клик на отдел → /tasks)
- **Квартальные планы по статусам** (список с цветными барами)

### 6. Зоны внимания
Компактный блок (≤5 пунктов): отделы с просроченными/критическими задачами, планы «Нужны правки», планы «На согласовании». Клик → переход в нужный раздел.

## Квартальные планы

Раздел `/plans` — матрица **отдел × квартал**.

### Состояние каждой ячейки
- **Бэклог** (серая плашка) — план не создан. Кнопка `+ Создать` сразу открывает диалог
- **В процессе** (синий) — черновик, ещё не отправлен
- **На согласовании** (жёлтый) — отправлен супер-админу/админу
- **Нужны правки** (красный) — возвращён с комментарием
- **Готов** (зелёный) — утверждён
- **Выполняется** (синий) — задачи в работе
- **Завершён** (зелёный) — итоги добавлены, период закрыт
- **Архивный** (серый) — закрытый период

В заполненной ячейке: статус-пилл, прогресс-бар, % выполнения, count `завершено/всего`, чипы «просроч.», «блок.», «риск».

**Клик на ячейку** → детальная страница плана.

### Создание плана
Кнопка `+ Создать` в пустой ячейке открывает диалог:
- Отдел, квартал — заполнены автоматически
- **Описание / цели на квартал** — свободный текст. Можно дополнить позже
- После создания план получает статус «В процессе» и сразу открывается

### Детальная страница плана
URL: `/plans/:id`

Шапка:
- Название отдела + квартал
- Статус-пилл, руководитель, автор, время последнего обновления
- Кнопки действий (зависят от роли и статуса):

| Кнопка | Кто видит | Когда |
|---|---|---|
| **Отправить на согласование** | Владелец / Рук. отдела | Статус `Черновик` или `Нужны правки` |
| **Утвердить** | Админ / Супер-админ | Статус `На согласовании` |
| **Вернуть на доработку** | Админ / Супер-админ | Статус `На согласовании`. **Комментарий обязателен** |
| **В работу** | Владелец / Рук. отдела | Статус `Готов` |
| **Завершить** | Владелец / Админ | Любой не-terminal статус. **Итоговый комментарий обязателен** |

Под шапкой — **6 карточек статистики**: Прогресс / Всего / Завершено / В работе / Под риском / Просрочено

Тело — **4 вкладки**:
- **Задачи** — карточки задач, привязанных к плану. Кнопка `+ Добавить задачу` открывает форму с предзаполненными отделом, кварталом и `plan_id`
- **Комментарии** — переписка по плану. Цветные выделения для системных событий (отправка, утверждение, запрос правок). ⌘+Enter / Ctrl+Enter — отправить
- **История** — таймлайн всех действий: смена статусов, изменения, комментарии. Объединено из `audit_log` и `status_change` записей
- **Итоги** — финальный обзор по кварталу. Заполняется при завершении плана или дополняется вручную

### Workflow согласования
1. Руководитель отдела создаёт план в Бэклоге → статус «В процессе»
2. Заполняет задачи и описание
3. Нажимает «Отправить на согласование» → статус «На согласовании», супер-админам приходит уведомление
4. Супер-админ открывает план:
   - **Утверждает** → статус «Готов», руководителю приходит нотификация
   - **Возвращает на доработку** → обязательно пишет комментарий → статус «Нужны правки», руководителю приходит нотификация
5. Руководитель дорабатывает и отправляет повторно
6. После утверждения нажимается «В работу» → статус «Выполняется»
7. По завершении квартала — кнопка «Завершить» с итоговым комментарием → статус «Завершён»

## Планы и задачи

Раздел `/tasks` — единое представление всех задач компании.

В шапке — **переключатель вида**:
- **Таблица** — для анализа, фильтров, сортировки
- **Канбан** — для движения задач по статусам

Под переключателем — панель фильтров:
- Поиск по названию/описанию/исполнителю
- Отдел
- Квартал (по умолчанию **текущий**)
- Статус (Все / Активные / Просроченные / + конкретные статусы)
- Приоритет
- Исполнитель
- Сортировка

### Вид «Таблица»
Колонки: Отдел, Задача, **Направление** (свободный текст), Квартал, Заказчик, Приоритет, Дедлайн, Статус, Исполнитель, Asana
- Просроченные дедлайны выделены красным
- Клик по строке → открывает диалог редактирования

### Вид «Канбан»
Колонки = статусы задач: **Бэклог**, **Запланировано**, **В работе**, **Под риском**, **Заблокирована**, **На проверке**, **Завершена**, **Отменена**
- **Любую карточку можно тащить** — клик на любое место карточки
- Клик без перетаскивания → открывает редактирование
- Перенос между колонками **мгновенный** (optimistic update — даже если сеть тормозит, карточка прыгает сразу)

### Создание задачи
Кнопка «**Новая задача**» в сайдбаре или шорткат `N`:

Поля:
- **Название** (обязательно)
- **Описание**
- **Отдел**, **Квартал**
- **Статус**, **Приоритет**
- **Дедлайн**, **Исполнитель** (выбор из списка зарегистрированных юзеров)
- **Заказчик**, **Ссылка Asana**
- **Заметки**
- **Направление** — свободный текст (ИИ, HR, Процессы, Отчётность). Локально для этой задачи, **не сохраняется** в общий справочник
- **Связанные цели** — KPI, к которым привязана задача. При статусе «Завершена» вклад автоматически прибавится к факту цели

## Цели (KPI)

Раздел `/kpi` — числовые метрики компании.

Каждая цель содержит:
- Название, описание
- Отдел, квартал
- **План** (целевое значение) и **Факт** (текущее, рассчитывается автоматически)
- Единица измерения (%, шт, ч, ...)
- Ответственный, дедлайн
- Теги (через глобальный TagPicker)

### Прогресс цели
- `Факт = (сумма прогресс-логов) + (сумма вкладов завершённых задач)`
- При достижении 100% — **запускается анимация конфетти** 🎉
- Цвет статуса: «Достигнуто» (зелёный) ≥100% / «В графике» ≥70% / «Под риском» ≥40% / «Отстаёт» <40%

### Детальная панель цели
Внутри карточки KPI — три вкладки:
- **Задачи** — связанные задачи и их вклад
- **Прогресс** — журнал ручных записей (`+N единиц`, дата, комментарий). ⌘+Enter — отправить
- **Чат** — комментарии. **Поддержка @-упоминаний**: набираешь `@`, выбираешь юзера → он получит нотификацию

## Отделы

Раздел `/directions` — список отделов.

Каждый отдел:
- Название, описание, цвет (для визуальной идентификации)
- Ответственный (текстовое поле)
- Список задач отдела
- Кнопка `+ Задача` создаёт задачу с предзаполненным `direction_id`

Создание/редактирование — через кнопки сверху и иконку «карандаш» на карточке.

> Назначение **главного руководителя** отдела (FK на пользователя) делается в **Управление → Отделы**.

## Архив

Раздел `/archive` — задачи, переведённые в архив.

В TaskDialog для существующей задачи есть кнопка **«В архив»** — она перемещает задачу в архив (поле `archived = true`). Из архива можно восстановить кнопкой «Восстановить».

> В будущем сюда же будут попадать закрытые квартальные планы.

## Управление

Раздел `/management` — доступен только админам и супер-админам. Шесть вкладок:

### Пользователи
Таблица всех пользователей системы:
- Имя, email, роль (inline-смена через Select), отделы (бейджи доступа), статус (Активен/Неактивен), регистрация, последняя активность
- Фильтры: поиск, роль, статус. Сортировка.
- Действия: ✏️ редактировать доступы (откроет диалог со списком отделов и уровнями) · ⏻ деактивировать/активировать · 🗑️ удалить

> Нельзя удалить себя. Админ не может назначить кого-то супер-админом — только супер-админ может это сделать.

### Роли и права доступа
Справочная страница: описание каждой роли + полная матрица возможностей.

### Отделы
Карточки всех отделов:
- Назначение **главного руководителя** через popover
- Количество пользователей с доступом, количество задач
- Создание/редактирование

### Кварталы
Управление периодами:
- **Создание квартала** — формат `Q1 2027`. Даты начала и конца проставляются автоматически
- **Inline-смена статуса**: Запланирован / Активен / Закрыт / Архивный
- **Видимость** (Виден / Скрыт) — скрытые кварталы не показываются обычным пользователям, но видны в Управлении и Архиве
- **Кнопка «Открыть»** для активных/запланированных кварталов → переход на рабочий экран `/quarters/:id`

### Матрица доступов
Таблица **пользователи × отделы**:
- Каждая ячейка — Select с уровнями: Нет доступа / Только просмотр / Просмотр + редактирование / Полный доступ
- Изменения сохраняются мгновенно
- Супер-админы и админы автоматически получают «Полный» во всех отделах (отображается серой плашкой)

### Журнал действий
Хронологическая лента всех административных действий: назначение/снятие ролей, выдача/изменение/отзыв доступов к отделам, смена руководителей, активация/деактивация пользователей, изменения статусов планов.

Каждая запись содержит: автора, действие, цель, краткое описание, относительное время.

## Уведомления

Колокольчик в нижней части сайдбара. Бейдж показывает количество непрочитанных.

### Типы уведомлений

**Задачи:**
- `task_assigned` — назначили задачу

**Цели (KPI):**
- `kpi_assigned` — назначили ответственным
- `kpi_mention` — упомянули в комментарии (через `@`)
- `kpi_comment` — добавили комментарий
- `kpi_progress` — добавили прогресс

**Квартальные планы:**
- `plan_on_review` — план отправлен на согласование (видят все админы и супер-админы)
- `plan_approved` — план утверждён
- `plan_changes_requested` — план возвращён на доработку
- `plan_at_risk` — план под риском
- `plan_blocked` — план заблокирован
- `plan_completed` — план завершён
- `plan_comment` — комментарий к плану

### Поведение
- **Realtime** — новые уведомления приходят мгновенно через WebSocket с тостом в правом нижнем углу
- **Self-уведомления** включены: даже если ты сам себя назначил — приходит нотификация (полезно как напоминание)
- Клик на уведомление → переход на связанную сущность + помечает прочитанным
- Кнопка «Прочитать» — пометить все как прочитанные
- Кнопка «Очистить все» — удалить всё

> Уведомления автоматически удаляются при удалении исходной сущности (задачи, цели, плана).

## Темы и горячие клавиши

### Темы
- Переключатель в сайдбаре (солнце ↔ луна)
- Тема сохраняется в localStorage
- По умолчанию — тёмная

### Горячие клавиши

| Клавиша | Действие |
|---|---|
| `⌘K` / `Ctrl+K` / `/` | Открыть Command Palette |
| `N` | Создать новую задачу |
| `G` затем `D` | Перейти на Панель управления |
| `G` затем `L` | Квартальные планы |
| `G` затем `T` | Планы и задачи |
| `G` затем `K` | Канбан-вид |
| `G` затем `P` | Цели |
| `G` затем `O` | Отделы |
| `G` затем `A` | Архив |
| `⌘+Enter` / `Ctrl+Enter` | Отправить комментарий в чате KPI / плана / прогрессе |

> Шорткаты не срабатывают, когда фокус в инпуте/textarea — не воруют ввод.

### Command Palette (⌘K)

Универсальный быстрый поиск:
- **Быстрые действия**: Новая задача, Новая цель
- **Навигация**: все разделы с шорткатами
- **Поиск задач** — fuzzy по названию/исполнителю/заказчику/отделу. Клик открывает редактирование
- **Поиск целей** — fuzzy по названию + текущий прогресс справа

## Частые сценарии

### Создать квартальный план для отдела
1. Открыть `/plans` → найти ячейку «отдел × квартал» с плашкой «Бэклог»
2. Нажать `+ Создать`
3. Заполнить описание/цели → «Создать»
4. На детальной странице: добавить задачи через `+ Добавить задачу`
5. Когда готов: «Отправить на согласование»

### Согласовать план (для админа)
1. В колокольчике (или дашборд → «Ожидают согласования») кликнуть на план
2. Изучить задачи и описание
3. Либо «Утвердить», либо «Вернуть на доработку» с обязательным комментарием
4. После согласования руководитель сможет нажать «В работу»

### Назначить задачу другому пользователю
1. Открыть задачу
2. В поле «Ответственный» выбрать пользователя из списка
3. Сохранить → ему придёт уведомление с ссылкой на задачу

### Сделать пользователя руководителем отдела
1. **Управление → Пользователи** → сменить роль на «Руководитель отдела»
2. **Управление → Отделы** → клик на «Руководитель» в карточке отдела → выбрать пользователя
3. **Управление → Матрица доступов** (опционально) → выдать «Полный доступ» к нужным отделам

### Закрыть квартал
1. **Управление → Кварталы** → сменить статус на «Закрыт» или «Архивный»
2. На рабочем экране каждого плана — нажать «Завершить» с итоговым комментарием

### Найти все критические задачи
- Дашборд → клик на карточку «Критический» в блоке «По приоритетам»
- Или Command Palette (⌘K) → «Планы и задачи» → фильтр Приоритет = Критический

### Привязать задачу к цели (KPI)
1. Открыть задачу
2. Внизу формы — блок «Связанные цели» → выбрать цель из списка → указать вклад → `+`
3. Когда задача станет «Завершена», вклад автоматически прибавится к факту цели

### @-упомянуть коллегу в чате цели
1. Открыть KPI → вкладка «Чат»
2. Набрать `@` → выпадет список юзеров → выбрать → имя вставится
3. Отправить → коллега получит уведомление со ссылкой на цель

## Решение проблем

### «Не вижу раздел Управление»
Раздел доступен только **админам и супер-админам**. Попроси супер-админа поднять тебе роль.

### «Метрики на дашборде расходятся с фактическими данными»
Скорее всего фильтры в шапке. Нажми **«Сброс»** или поставь «Все кварталы».

### «Создал задачу — она не появляется на дашборде»
Проверь её квартал. По умолчанию дашборд показывает только **текущий квартал**. Переключи в шапке на нужный.

### «Уведомления зависли — задачу удалил, нотификация осталась»
- Уведомления удаляются автоматически при удалении сущности (триггеры на стороне БД).
- Если что-то осталось — открой колокольчик и нажми «Очистить все».

### «Не работает реалтайм — новые комментарии не появляются»
Хард-рефреш страницы (Cmd+Shift+R / Ctrl+Shift+R) — это пересоздаст подписку.

### «Поле "Направление" пустое в каждой задаче»
Это **намеренно**. Поле локальное для каждой задачи, заполняется вручную. Никакого автозаполнения из других задач.

### «Не могу удалить пользователя — кнопка серая»
Нельзя удалить себя. Попроси другого админа.

### «Получил уведомление, кликнул — оно ведёт в никуда»
Возможно сущность была удалена, а уведомление осталось. Триггер очистки сработает в следующий раз. Можно вручную через «Очистить все».

### «Кварталы 2025 не показываются»
По дизайну. Они скрыты (`is_visible=false`). Чтобы вернуть — **Управление → Кварталы** → нажать на «Скрыт» рядом с кварталом.
